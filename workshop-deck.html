01 · Conceptos base

Un LLM predice texto.
No piensa como tu.

  • →Fue entrenado con grandes cantidades de texto para adivinar que viene despues
  • →Reconoce patrones. No entiende el mundo como una persona
  • →Por eso puede sonar brillante y equivocarse con mucha seguridad
  • →Piensalo como un asistente nuevo: rapido, util y sin contexto de tu vida
01 · Conceptos base

Cuatro conceptos para no perderte

Modelo
El motor que genera la respuesta. Distintos modelos priorizan velocidad, calidad o especializacion.
Prompt
Lo que le pides. Instrucciones, contexto, restricciones y el tipo de resultado que esperas.
Token
Pedacitos de texto que el modelo procesa. Afectan costo, velocidad y cuanta informacion cabe.
Contexto
La informacion que el modelo tiene a mano para resolver tu tarea en ese momento.
01 · Conceptos base

La IA es util, pero tiene limites reales

  • →Puede inventar datos, fuentes o detalles cuando no sabe
  • →No siempre tiene informacion actualizada si no puede buscar en la web
  • →No conoce tu contexto personal o laboral a menos que se lo des
  • →Sirve excelente para explorar, resumir y redactar; no para confiar ciegamente

02 · Herramientas

Empieza por la tarea.
Luego elige la herramienta.

  • →Si necesitas actualidad, usa una herramienta con web y fuentes visibles
  • →Si necesitas claridad o borradores, empieza por un chat generalista
  • →Si necesitas estructura, pide tablas, checklist o pasos concretos
  • →Si una tarea se repite mucho, guarda un prompt reutilizable
Regla simple
No te cases con una herramienta. Compara dos cuando el resultado importe.
03 · Prompting

Promptear bien es dar contexto.
No usar palabras magicas.

  • →Una instruccion vaga produce una respuesta vaga
  • →La IA responde mejor cuando entiende objetivo, audiencia y restricciones
  • →Tratalo como un brief para alguien inteligente que no conoce tu caso
  • →Una linea extra de contexto puede ahorrarte varios intentos malos

03 · Prompting

Cinco habitos que mejoran cualquier resultado

  • →Se especifico. Mejor breve y claro que largo y ambiguo
  • →Divide tareas complejas. Primero esquema, luego detalle
  • →Pide formato. Tabla, bullets, pasos, resumen, email
  • →Itera. Mejorar en dos turnos casi siempre gana
  • →Verifica. Nombres, numeros, fechas y fuentes antes de usar
05 · Workflows

No tienes que cambiar tu vida.
Empieza con un workflow pequeño.

  • →Resume un texto largo antes de leerlo completo
  • →Convierte notas desordenadas en una lista de accion
  • →Transforma un borrador flojo en una version mas clara
  • →Usa IA al inicio para pensar y al final para pulir
05 · Workflows

Delega la friccion.
Conserva el criterio.

  • →La IA puede proponer, ordenar, resumir y redactar
  • →Tu decides objetivo, contexto y la version final
  • →Si el error cuesta caro, revisa manualmente antes de usar
  • →Human-in-the-loop no es desconfianza: es buen diseno de trabajo
06 · Seguridad

Si no lo pondrias en un email abierto, no lo pegues sin pensar

  • →Evita pegar datos sensibles, credenciales, contratos o informacion privada
  • →Anonimiza nombres y datos cuando solo necesitas trabajar la estructura
  • →Asume que lo que subes puede quedar guardado por el proveedor
  • →Si trabajas con informacion delicada, sigue la politica de tu equipo
