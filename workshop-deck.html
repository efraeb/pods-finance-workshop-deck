01 · AI primitives

An LLM is a text prediction engine,
not a thinking machine

  • →Trained on massive text. Predicts what comes next.
  • →Statistical patterns, not understanding
  • →That's why it can be brilliant and confidently wrong in the same sentence
  • →Think of it as a new hire who read the entire internet but knows nothing about your company
01 · AI primitives

Four concepts you need to know

Token
The basic unit an LLM reads and writes. Roughly one word or word fragment. Everything you send and receive is measured in tokens. The entire Harry Potter series = 1.1M tokens.
Prompt
Your input to the model: the instruction, question, or context you provide. The quality of your prompt determines the quality of the output.
Context window
How much the model can hold in working memory at once. Claude's window fits 200K tokens (~500 pages). Longer conversations degrade recall. This is "context rot."
Hallucination
When the model generates something that sounds confident and correct but is fabricated. It doesn't know it's wrong. Always verify critical facts.

03 · Context management

Long chat? Don't delete.
Compact it.

  • →The instinct is to start fresh or delete old messages. Don't.
  • →Dropping messages wipes context the model was using, you get worse output without knowing why.
  • →Better move: ask the model to summarize the conversation so far, then continue.
  • →This keeps what matters and frees up space. Claude calls this compaction.

02 · Prompt engineering

Five tips that improve any prompt

  • →Be specific. Not "write an email" but "5-sentence follow-up, optimistic tone, no smart contract jargon"
  • →Break complex tasks into steps. Ask for the outline first, then each section separately
  • →"Think step by step" triggers deeper reasoning. Fewer errors on multi-variable problems.
  • →Request your format. Table, JSON, bullets, markdown. If you don't say, the model guesses.
  • →Iterate, don't perfect. A decent prompt refined in 3 turns beats chasing the perfect first shot.
02 · Prompt engineering

AI multiplies your output.
Your judgment is still the input.

  • →The most reliable setup is AI drafting, a human reviewing
  • →No single model does everything. Combine tools.
  • →This workshop was built with AI, but a human reviewed every section, adjusted the tone, and removed what didn't apply
  • →Today it's Claude, tomorrow it could be something else. Don't marry a tool.
Good prompting gets you most of the way there. But every new chat, the AI starts from zero, no memory of who you are, what you do, or how you communicate. There's a better solution.

04 · Agent orchestration

A chat answers once. An agent figures out the steps itself.

  • →Chat: you ask, it answers, it waits for you. One turn.
  • →Agent: LLM + context + tools + a loop. Takes a step, sees the result, decides what's next.
  • →The loop is the difference. It runs until the task is done, or until it hits a wall and needs your call.
  • →Analogy: chat = asking a colleague a quick question. Agent = delegating a project to a colleague who figures out the steps.
04 · Agent orchestration

AI Native Tools

  • →Web search: browse the internet, pull live data, not just training knowledge
  • →Code execution: write and run code, process spreadsheets, calculate
  • →File reading and writing: work with documents you provide
  • →Image analysis: describe, compare, extract data from visuals
No developer required
Claude on claude.ai already has most of these. You don't need a developer to start. Open claude.ai and try it after this session.
04 · Agent orchestration

Start simple. Add complexity only when you hit a wall.

LevelWhat it isWhen to use it
Copy-paste prompt A well-written prompt you reuse 80% of your daily work
Scheduled prompt Same prompt, runs on a timer Daily summaries, weekly reports, recurring checks
Single agent + tools LLM + native tools, multi-step Research + draft tasks, anything with multiple steps
Orchestrator Multiple agents in parallel Complex pipelines: research + draft + review simultaneously

The best automation is the one you actually use. Don't over-engineer.

04 · Agent orchestration

Human-in-the-loop is a design decision, not a philosophy

  • →When you build an agent workflow, you're choosing where it runs on autopilot and where it pauses for you
  • →The agent handles the grunt work. The human approves at the decision points.
  • →The most reliable systems aren't fully autonomous. They have checkpoints
  • →Design question: "At which steps does human judgment change the outcome?" Those are your checkpoints.
You know what agents can do in theory. The real question is what you should actually hand off. Not all the Twitter hype holds up yet. Let's talk about where it actually works.

05 · Security & reliability

If you wouldn't post it on your X feed, don't paste it into an AI tool

  • →AI providers can see your chat history
  • →Once you paste sensitive data, it's no longer yours alone
  • →This applies to all tools. Claude, ChatGPT, Gemini, Copilot. Same risk.
